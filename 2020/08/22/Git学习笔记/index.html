基本概念

Git的四种工作区域

  • WorkSpace(工作区)：本地看到的文件和目录，受Git管理
  • Index(暂存区)：未提交的改动
  • Repository(本地仓库)：暂存区的文件提交后会存放到本地仓库
  • Remote(远程仓库)：类似Github之类的

Git的工作流程

远程仓库克隆代码->工作目录修改代码->改动添加到暂存区->提交到本地仓库->推送到远程仓库

Git管理的文件的四种状态

  • unchecked：文件未被git管理
  • unmodified：未被修改
  • modified：文件被修改过
  • stage：暂存状态，修改后未被提交的状态，提交后变为unmodified
