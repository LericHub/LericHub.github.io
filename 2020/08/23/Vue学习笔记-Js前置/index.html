Vue学习笔记-Js前置

|阅读量:

js的学习代码看这里 【https://gitee.com/Lyric-git/learning-notes/tree/master/Vue/Vue前置】

变量

变量类型

  • Numbers，String，Boolean
  • Object，Array，Undifined 称为对象类型

Javascript 在调用前需要先声明

  • var：可以重复声明
  • let：只能一次声明，可以重新赋值
  • const：常量，声明后不能修改值，但是对于数组可以对元素的数量进行修改：掐头去尾

查看变量的类型：typeof

函数以及自调用函数

定义一个函数

箭头函数

箭头函数相当于匿名函数，且必须是匿名函数

函数体内只有一行，无需大括号

取代return–如果要加return，则必须大括号

自调用函数

在声明结束后马上调用的函数

对于函数来说它要么要有个函数名要么要有承接它的变量

在调用函数时，函数名或者承载它的变量可以理解为函数的方法体，而括号可以理解为执行这个方法

自调用函数只需要在函数声明之后加上一个括号表示执行

this的指向问题

默认绑定

在全局环境下this指向window

函数在独立调用时，this指向window，可以理解为是window调用了这个函数

函数在最为对象的属性被调用时，this指向调用它的对象

IIFE自执行函数(相当于独立调用),内部的this指向window

隐式绑定

隐式绑定就是通过调用函数，使被调用的函数与对象进行绑定

而函数内的this永远指向调用它的对象

隐式丢失

隐式绑定对导致隐式丢失的问题，即通过隐式绑定的函数丢失绑定对象，默认绑定到window的现象

主要原因就是在传递的过程中传入的参数是函数体，从而失去原有的绑定对象

  1. 函数内的方法被赋值给变量，出现隐式丢失

    把 obj.foo() 赋值给别名bar,造成隐式丢失，因为此时 bar 只是function foo() { 方法体 }

  1. 当函数内的方法被当作参数,传递后,会出现隐式丢失
  1. 内置函数导致的隐式丢失
  1. 间接调用导致隐式丢失

显式绑定

显式绑定就是通过 call( ), apply( ), bind( ) 函数将函数体插入到对象内，实现绑定

  • call( ) apply( )：无返回值，绑定的效果是暂时的
    • call( ) 只能插入到对象内
    • appl( ) 能插入到对象或者数组内
  • bind( ) ：有返回值，相当于拷贝了被当顶对象的副本再将方法插入拷贝里面去
  1. 显式绑定
  1. 硬绑定
  1. foreach()函数

new绑定

当new关键字来执行函数，相当于构造函数来实例化对象，因此this指向当前实例化的对象

严格模式的this指向

  1. 当使用严格模式的时候，this默认为undifined即原来只想window的是undifined
  1. 严格模式，函数apply()和call()内部的this始终是他们的第一个参数,给什么是什么
